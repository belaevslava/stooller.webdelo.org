Google services

  1. Directions API

    Сервис дает нам информацию о расстоянии, времени пути и пробках. На это соновании мы расчитываем расход топлива для выбранного маршрута и формируем цену транспортировки.

  2. Places API

    Для удобства клиентов, маркеры выводятся по точным адресам переезда. По полученым координатам определяется зона обслужения сервисом.

  3. Calendar API

    В праздничный день переезд будет стоить дороже. Calendar API позволяет определить является ли выбранный день праздничным.

  4. Polygon API

    Зоны обслуживания сервиса выделены полигоном. Polygon API дает возможность убедиться, что выбранные адреса находятся в зоне обслуживания.

  5. Autocomplete API

    Мы получаем точный адрес С целью точнее определить положение адресов переезда, для ввода адреса используется сервис. Это помогает также избежать ошибок.

Amazon Web Services

  1. Amazon EC2

    Сервис Stooller обслуживает клиентов на двух континентах. От физического расположения сервера, зависит скорость доступа клиентов. После правильной настройки Amazon EC2 все клиенты получают лучший доступ к сервису, независимо от своего географического положения.

  2. Amazon SES

    Для повышения эффективности e-mail рассылки, настроен сервис Amazon SES. Это техническое решение повывает надежность рассылки, масштабируемость.

  3. Amazon RDS

    Надежность в хранении данных и высокая скорость доступа к данным были одним из главных приоритетов при разработке Stooller. После настройки Amazon RDS сервис получил масштабируемую и быструю Базу Данных.
